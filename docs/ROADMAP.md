# 제품화 로드맵

## Phase 0 — 현재 저장소

- 플레이 가능한 레드 존 vertical slice
- 적응형 웨이브와 자연어 전술 명령
- 로컬 저장, 오프라인 생산, 모집/천장
- 반응형 PC/모바일 입력
- 20Hz 권위형 게임룸, 서버 입력 검증, 상태 동기화
- JWT 게스트 계정, PostgreSQL 경제 저장, Redis 수평 확장
- Docker Compose 개발 환경, 통합 테스트와 CI
- 전용 적·자원 실루엣, 전투 VFX, 합성 SFX와 진동 피드백
- 첫 실행 온보딩, 사운드·진동·모션 감소 설정
- PWA 설치 자산과 Capacitor Android 내부 테스트 프로젝트
- 3종 무장과 서버 공용 전투 규칙, 5단계 첫 작전과 케르베로스 보스전
- 투명한 보급소 UI, 서버 결제 검증 경계와 전역 영수증 중복 방지
- 핵심 전환 퍼널의 PostgreSQL 이벤트 저장과 월 매출 실험 기준
- 일러스트형 전장 오퍼레이터, 역할별 뉴럴 링크와 보스·필살기 컷인
- 게임패드, 서버 검증 회피, 모바일 수동 추출과 접근성 설정
- 선택 분석 동의, 데이터 내보내기, 계정 삭제와 AI 처리 고지
- 뉴럴 재머 적성 개체와 서버 권위형 링크 교란
- 비공개 알파 진단 화면, 동의 기반 오류 큐와 서버 준비 상태 검사
- Android 서명 AAB 워크플로와 다중 클라이언트 알파 부하 스모크
- 두 번째 바이옴형 작전 OP-01 ASHFALL, 신경 중계기와 다중 패턴 헤카톤 보스
- 서버 저장형 캠페인 해금, 작전별 매칭과 최초 완료 보상
- 작전별 결정론적 하드 커버, 공용 이동·회피·사선 판정과 적 엄폐 우회
- 추출 재화 기반 전술 장비 4종, 워크숍 제작과 서버 저장형 2슬롯 로드아웃
- 동일 세션 자동 재접속, 새 세션 복구와 단절 시 안전한 로컬 폴백
- FPS 기반 자동 그래픽 품질, HUD 스로틀과 전투 이펙트 풀링
- 최대 128클라이언트·1시간 구성형 서버 소크 테스트
- 7개 오퍼레이터의 공유 페르소나 설정과 캐릭터별 딥 토크·기억 관리 UI
- 명시적 동의·입출력 안전 검사·일일 비용 상한을 갖춘 서버 중계형 Responses API
- 제공자 장애·타임아웃·할당량 초과에서도 동작하는 결정론적 페르소나 폴백

## Phase 1 — 6~10주 프리프로덕션

- 오퍼레이터 역할별 실제 스킬과 세 번째 작전 보스 패턴 확장
- 절차 생성 맵 청크와 파괴 가능한 구조물, 슬롯형 파밍 인벤토리와 장비 티어 확장
- Android 실기기 성능·발열 테스트와 Steam 데스크톱 빌드
- Steam·Google·Apple 계정 연결과 게스트 계정 이전

## Phase 2 — 3~5개월 알파

- 한국어/영어 STT·TTS와 전술 의도용 서버 AI 오케스트레이터
- 장기 기억 직접 수정·병합과 안전 신고 UI
- 8명의 오퍼레이터와 개인 스토리
- 절차 생성 임무 템플릿 20종, 바이옴 2종
- 협동 PvE 네트워크 검증, 안티치트 기초
- 결제 없는 코어 모집 밸런스 테스트

## Phase 3 — 6~9개월 소프트 런치

- Android 제한 지역 테스트 및 Steam 비공개 플레이테스트
- 확률 공개·연령·환불·개인정보·AI 고지 체계
- 코스메틱/시즌 패스 A/B 테스트
- AI 호출 비용, 캐시율, 안전 신고, 리텐션 기반 운영 대시보드
- 저사양 모바일 발열·메모리·네트워크 최적화

## Phase 4 — 출시 후보

- 콘텐츠: 바이옴 4+, 적 팩션 3, 오퍼레이터 20+, 엔드게임 레이드
- 플랫폼별 심의, 스토어 SDK, 업적/클라우드 저장/푸시
- 데이터 이전, 장애 복구, 지역별 서버와 고객지원 체계
- 외부 보안 감사와 확률/경제 독립 검증

## 현재 출시 판정

v1.2는 웹/PWA와 Android에서 두 개 작전의 보스·추출·캠페인 해금, `탐사 → 추출 → 제작 → 재투입` 성장과 캐릭터별 딥 토크를 검증할 수 있는 **9점대 초반 프라이빗 알파 후보**입니다. 동의 기반 외부 AI 중계, 안전·비용 상한과 규칙 폴백까지 코드·모의 제공자 테스트로 검증했습니다. 영수증 원장과 검증 경계는 준비됐지만 실제 플랫폼 결제 어댑터가 없으므로 알파에서는 보급소를 숨깁니다. 공개 스토어 제출 전에는 네이티브 결제·계정 복구, 운영자 정보가 포함된 공개 정책 URL, 연령 등급, 외부 AI 실키 평가·레드팀, 실기기 성능·발열 QA와 고객지원 채널을 완료해야 합니다.

## 기술적 결정 게이트

현재 Phaser vertical slice는 시장성과 코어 루프 검증에 적합합니다. 다음 조건 중 둘 이상이 확정되면 Godot/Unity 3D 클라이언트 전환을 결정합니다.

- 자유 시점 3D 사격이 제품의 필수 차별점으로 검증됨
- 4인 이상의 실시간 협동과 복잡한 물리 상호작용이 필수
- 콘솔 출시 또는 고급 3D 애니메이션 파이프라인이 확정됨
- 모바일 성능 목표를 웹 런타임에서 충족하지 못함

AI 서비스, 저장 스키마, 전술 명령 계약은 엔진과 분리하여 전환 비용을 낮춥니다.
