# 제품화 로드맵

## Phase 0 — 현재 저장소

- 플레이 가능한 레드 존 vertical slice
- 적응형 웨이브와 자연어 전술 명령
- 로컬 저장, 오프라인 생산, 모집/천장
- 반응형 PC/모바일 입력
- 20Hz 권위형 게임룸, 서버 입력 검증, 상태 동기화
- JWT 게스트 계정, PostgreSQL 경제 저장, Redis 수평 확장
- Docker Compose 개발 환경, 통합 테스트와 CI
- 전용 적·자원 실루엣, 전투 VFX, 합성 SFX와 진동 피드백
- 첫 실행 온보딩, 사운드·진동·모션 감소 설정
- PWA 설치 자산과 Capacitor Android 내부 테스트 프로젝트

## Phase 1 — 6~10주 프리프로덕션

- 게임패드, 모바일 수동 추출 버튼, 색각 보조와 UI 배율 옵션
- 무기 3종, 오퍼레이터 역할별 실제 스킬, 보스 1종
- 맵 청크와 구조물 충돌, 파밍 인벤토리, 장비 제작
- Android 실기기 성능·발열 테스트와 Steam 데스크톱 빌드
- Steam·Google·Apple 계정 연결과 게스트 계정 이전

## Phase 2 — 3~5개월 알파

- 서버 AI 오케스트레이터, 한국어/영어 STT·TTS
- 동의 기반 장기 기억 조회·수정·삭제 UI
- 8명의 오퍼레이터와 개인 스토리
- 절차 생성 임무 템플릿 20종, 바이옴 2종
- 협동 PvE 네트워크 검증, 안티치트 기초
- 결제 없는 코어 모집 밸런스 테스트

## Phase 3 — 6~9개월 소프트 런치

- Android 제한 지역 테스트 및 Steam 비공개 플레이테스트
- 확률 공개·연령·환불·개인정보·AI 고지 체계
- 코스메틱/시즌 패스 A/B 테스트
- AI 호출 비용, 캐시율, 안전 신고, 리텐션 기반 운영 대시보드
- 저사양 모바일 발열·메모리·네트워크 최적화

## Phase 4 — 출시 후보

- 콘텐츠: 바이옴 4+, 적 팩션 3, 오퍼레이터 20+, 엔드게임 레이드
- 플랫폼별 심의, 스토어 SDK, 업적/클라우드 저장/푸시
- 데이터 이전, 장애 복구, 지역별 서버와 고객지원 체계
- 외부 보안 감사와 확률/경제 독립 검증

## 현재 출시 판정

v0.4는 웹/PWA와 Android에서 코어 루프를 검증할 수 있는 **내부 테스트용 vertical slice**입니다. 스토어 제출용 제품으로 판정하려면 Phase 1의 콘텐츠 깊이와 실기기 최적화에 더해, 네이티브 결제·계정 복구·개인정보 동의·연령 등급·AI 고지·확률 공개·크래시 수집·고객지원 체계를 먼저 완료해야 합니다.

## 기술적 결정 게이트

현재 Phaser vertical slice는 시장성과 코어 루프 검증에 적합합니다. 다음 조건 중 둘 이상이 확정되면 Godot/Unity 3D 클라이언트 전환을 결정합니다.

- 자유 시점 3D 사격이 제품의 필수 차별점으로 검증됨
- 4인 이상의 실시간 협동과 복잡한 물리 상호작용이 필수
- 콘솔 출시 또는 고급 3D 애니메이션 파이프라인이 확정됨
- 모바일 성능 목표를 웹 런타임에서 충족하지 못함

AI 서비스, 저장 스키마, 전술 명령 계약은 엔진과 분리하여 전환 비용을 낮춥니다.
