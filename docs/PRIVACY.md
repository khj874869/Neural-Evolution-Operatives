# 개인정보·AI 처리 안내 초안

최종 수정: 2026-07-24

이 문서는 `Neural Evolution: Operatives` v1.3에 구현된 데이터 흐름을 설명하는 제품 초안입니다. 공개 배포 전 운영 주체명, 연락처, 사업장 소재지, 국외 이전 대상·재위탁사와 실제 보유 기간을 확정해 법률 검토를 받아야 합니다.

## 처리하는 데이터

- 필수: 무작위 게스트 식별자, 게임 진행도, 재화, 쉘터, 오퍼레이터, 분대 편성, 일일·주간 계약 진행과 연속 달성 기록, 서버 전투·경제 처리 기록
- 거래 안전: 플랫폼, 거래 식별자, 상품, 구매 시각, 금액과 통화. 중복 지급과 환불 대응에 사용
- 선택 분석: 사용자가 동의한 경우에만 튜토리얼·작전·보급소 진행과 제한된 오류 종류를 기록
- 선택 AI: 사용자가 외부 AI 링크에 별도 동의하고 대화를 보낼 때 입력 원문, 선택한 오퍼레이터 설정, 호감도와 최근 요약 기억을 응답 생성과 안전 검사를 위해 처리

채팅·전술 명령 원문과 AI 대화 원문은 분석 이벤트에 기록하지 않습니다. 마이크 음성, 주소록, 위치, 광고 식별자도 수집하지 않습니다. 오류 메시지는 URL과 UUID 형태 식별자를 제거하고 스택 없이 전송하며, 동일 지문은 30초 동안 중복 억제합니다.

## AI 처리

전술 명령은 허용된 명령 집합으로 기기와 게임 서버에서 처리하며 외부 LLM이 전투 결과를 결정하지 않습니다. 딥 토크는 기본적으로 규칙 기반 페르소나 코어를 사용합니다.

사용자가 설정에서 외부 AI 링크를 켜고 개별 메시지를 전송한 경우에만 게임 서버가 OpenAI API로 다음 정보를 중계합니다.

- 입력한 메시지 원문
- 선택한 오퍼레이터의 공개 설정과 말투
- 0~100 범위 호감도와 최근 요약 기억 최대 6개
- 원본 게스트 ID가 아닌 단방향 해시 기반 안전 식별자

서버는 입력·출력을 `omni-moderation-latest`로 검사하고 Responses 요청에 `store: false`를 지정합니다. 이는 모델 응답 저장을 요청하지 않는 애플리케이션 설정이며, 제공자의 보안·오남용 방지 처리와 실제 보유 조건은 배포 계정 계약 및 제공자 정책을 따릅니다. 공개 배포 전 국외 이전 고지, 처리 위치, 보유 기간과 삭제 요청 경로를 실제 계약에 맞춰 확정해야 합니다.

게임 서버는 전체 입력 원문을 프로필에 저장하지 않습니다. 대화 후에는 최대 72자의 입력에서 만든 요약형 기억 문장, 마지막 오퍼레이터 응답, 처리 방식(`ai` 또는 `rules`), 시각과 일일 사용량을 저장합니다. 안전 필터로 차단된 입력은 기억에 저장하지 않습니다. 세션 화면의 대화 기록은 앱 메모리에만 있으며 다시 시작하면 사라집니다.

## 선택과 권리

- 첫 실행에서 선택 분석을 거부해도 모든 핵심 게임 기능을 사용할 수 있습니다.
- 설정의 `개인정보·AI 안내`에서 선택 분석과 외부 AI 링크를 각각 켜거나 끌 수 있습니다.
- 외부 AI 링크를 거부하거나 철회해도 규칙 기반 딥 토크와 모든 전투 기능을 사용할 수 있습니다.
- 오퍼레이터 딥 토크 화면에서 해당 캐릭터의 요약 기억과 마지막 응답을 삭제할 수 있습니다.
- 같은 화면에서 서버 또는 로컬 진행 데이터를 JSON으로 내보낼 수 있습니다.
- `DELETE`를 직접 입력한 후 계정을 삭제할 수 있습니다. 진행도·분석·경제 이벤트는 삭제됩니다.
- 거래 식별자는 중복 지급 방지와 법적 의무 이행을 위해 계정과 분리해 제한적으로 보존될 수 있습니다.

## 보안과 보유

게임 서버는 TLS가 적용된 배포 환경을 전제로 하며, JWT 인증·입력 길이 검증·중복 실행 방지·분당 요청 제한·일일 외부 AI 상한·PostgreSQL 트랜잭션을 사용합니다. 제공자 키는 서버 환경변수에만 두며 클라이언트 빌드에 포함하지 않습니다. 보유 기간은 공개 배포 전 운영 정책과 관할 법률에 맞춰 확정해야 합니다.
