# v1.2 Android 비공개 알파 운영서

## 목표

v1.2는 결제 전환이 아니라 두 작전의 안정성, 조작성, 서버 연결 품질과 선택형 딥 토크 만족도를 검증하는 빌드입니다. 보급소는 `VITE_ENABLE_COMMERCE=false`가 기본이며 실제 플랫폼 샌드박스와 영수증 검증기가 연결되기 전에는 노출하지 않습니다.

## 배포 전 필수 환경

- HTTPS 게임 서버와 PostgreSQL, Redis
- 32자 이상의 무작위 `JWT_SECRET`
- 실제 클라이언트 도메인만 허용한 `CORS_ORIGIN`
- `RELEASE_CHANNEL=alpha`와 배포 커밋의 `COMMIT_SHA`
- 공개 개인정보처리방침 URL과 테스터 문의·피드백 URL
- Google Play Console 앱과 내부 테스트 트랙
- 외부 AI 시험 시에만 서버 비밀 저장소의 `OPENAI_API_KEY`와 확정된 데이터 이전 고지

서버 배포 후 다음을 확인합니다.

```bash
curl -fsS https://SERVER/health
curl -fsS https://SERVER/ready
npm run smoke:server -- https://SERVER
npm run smoke:alpha -- https://SERVER 8 10000
npm run smoke:soak -- https://SERVER 16 120000
```

`/health`는 프로세스 생존, `/ready`는 저장소 연결 상태를 판정합니다. 기본 스모크는 장비 제작, 규칙 기반 딥 토크 기억 저장과 동일 세션 재접속까지 확인하고, 알파 부하 스모크는 기본 8개 클라이언트의 인증·게임룸 입장·상태 수신을 동시에 검사합니다. 소크 테스트는 최대 128클라이언트와 1시간까지 설정할 수 있으며 준비 상태 실패, 비정상 종료와 스냅샷 최대 간격을 함께 집계합니다.

딥 토크는 다음 두 구성으로 각각 확인합니다.

1. `OPENAI_API_KEY` 없이 규칙 기반 응답, 기억 저장·삭제와 계정 내보내기를 확인합니다.
2. 비공개 테스트 키를 서버에만 설정하고 동의 전 외부 호출 0건, 동의 후 안전 필터·일일 상한·타임아웃·로컬 폴백을 확인합니다.

차단 대상 입력은 프로필 기억에 남지 않아야 하며, 외부 AI 동의를 끈 뒤의 새 요청은 서버에 이전 동의가 잠시 남아 있어도 외부로 전송되지 않아야 합니다.

## Android 빌드

GitHub Actions의 `Android Private Alpha` 워크플로를 수동 실행하고 HTTPS 서버 URL과 선택 피드백 URL을 입력합니다.

서명 AAB를 만들려면 저장소 Actions secrets에 다음 값을 등록합니다.

- `ANDROID_KEYSTORE_BASE64`
- `ANDROID_KEYSTORE_PASSWORD`
- `ANDROID_KEY_ALIAS`
- `ANDROID_KEY_PASSWORD`

서명 정보가 없으면 워크플로는 설치 확인용 디버그 APK를 생성합니다. 서명 정보가 있으면 Play Console 내부 테스트용 release AAB를 생성합니다. 키스토어 원본과 비밀번호는 저장소에 커밋하지 않습니다.

## 실기기 매트릭스

최소 다음 여섯 조합에서 Operation Zero를 처음부터 끝까지 실행합니다.

| 등급 | 예시 조건 | 확인 항목 |
|---|---|---|
| 저사양 Android | 4GB RAM, 2021년 이전 SoC | 발열, 프레임 저하, 강제 종료 |
| 중급 Android | 6~8GB RAM | 기본 터치 조작과 재접속 |
| 고급 Android | 120Hz 디스플레이 | 프레임 페이싱과 배터리 사용 |
| 소형 화면 | 6인치 미만 또는 큰 글꼴 | HUD 겹침과 버튼 오입력 |
| 태블릿 | 10인치 이상 | 화면 비율과 터치 거리 |
| 게임패드 | 표준 XInput/Bluetooth | 이동·사격·회피·추출·뉴럴 링크 |

각 기기에서 신규 설치, 백그라운드 복귀, Wi-Fi↔모바일망 전환, 서버 강제 종료 후 복구를 한 번씩 검사합니다.

## 알파 통과 기준

- 자동 테스트와 웹·서버·Android 빌드 전부 성공
- 클라이언트·서버 자동 테스트 57개 통과
- 8개 동시 클라이언트 스모크에서 인증·입장 실패 0건
- 50회 Operation Zero 세션에서 진행 불가 오류 0건
- 크래시 없는 세션 99.5% 이상
- 튜토리얼 완료율 70% 이상
- 첫 작전 완료율 45% 이상
- P0/P1 버그 0건, 계정·재화 유실 0건

이 수치는 시장 벤치마크가 아니라 공개 테스트로 넘어가기 위한 내부 품질 게이트입니다.

## 테스터 제보 형식

게임의 `알파 정보`에서 진단 정보를 복사한 뒤 다음 항목과 함께 제출합니다.

1. 발생 시각과 기기 모델·OS
2. 재현 순서
3. 기대 결과와 실제 결과
4. 스크린샷 또는 15초 내외 영상
5. 복사한 빌드·서버 진단 정보

선택 분석을 허용한 경우 오류 메시지는 URL·식별자·스택을 제거하고 동일 오류를 30초 동안 중복 억제해 서버에 저장합니다. 대화 원문은 분석 이벤트에 저장하지 않습니다. 외부 AI 링크에 별도 동의하고 전송한 딥 토크 원문은 응답 생성과 안전 검사를 위해 제공자에게 중계될 수 있습니다.

## 롤백

진행 불가, 계정 유실, 반복 결제, 서버 과부하가 확인되면 Play Console 배포 비율을 즉시 0%로 내리고 이전 AAB로 되돌립니다. 서버는 이전 컨테이너 이미지로 롤백하되 데이터베이스 백업을 먼저 보존합니다.
