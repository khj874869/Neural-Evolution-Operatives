# NEURAL EVOLUTION: OPERATIVES

> 포스트 아포칼립스 오픈월드 생존 슈팅과 AI 오퍼레이터 수집·쉘터 성장을 결합한 PC/모바일 크로스플랫폼 v1.2 private alpha

마더브레인에 의해 문명의 90%가 붕괴된 이후, 플레이어는 자아를 되찾은 전술 오퍼레이터와 함께 레드 존을 탐사하고 지하 쉘터를 재건합니다. 현재 저장소에는 **Android 비공개 테스트와 웹 플레이테스트를 위한 v1.2 private alpha**가 들어 있습니다.

## 지금 플레이할 수 있는 기능

- 2,400 × 2,400 레드 존 탑다운 슈팅 및 자원 파밍
- 작전별 16개 하드 커버, 벽 슬라이딩, 회피 관통 방지와 엄폐 사격 차단
- 3명의 오퍼레이터 동료 자동 전투와 자연어 전술 명령
- 명중률·정지 시간·이동량·피해량을 분석하는 적응형 AI 디렉터
- 드론, 약탈자, 스토커, 브레이커, 뉴럴 재머와 애시폴 공병의 차별화된 일반 적
- 방사능/산성비/한파 콘셉트의 환경 재해 루프
- 현장 화물을 중앙 추출 지점으로 가져오는 생존형 리스크/리워드
- 쉘터 4개 모듈 업그레이드와 최대 8시간 오프라인 생산
- R/SR/SSR 오퍼레이터 수집, 중복 성장, 20회 SSR 천장
- 세라·모로·래칫·엠버·루멘·룩·패치 7명의 개별 캐릭터 일러스트
- 일러스트 중심 오퍼레이터 상세 화면과 서버 저장형 3인 분대 편성

- 분대 조합에 따른 화력·연사·기동·방사선 저항·재생·회수 보너스
- 추출 재화로 제작하는 전술 장비 4종, 2슬롯 로드아웃과 전장 HUD

- 입력·출력 안전 필터, 8초 타임아웃, 분당 요청 제한과 제공자 장애 시 규칙 기반 폴백

- 클라이언트 예측과 서버 재검증을 결합한 1.8초 쿨다운 회피
- Colyseus 기반 20Hz 서버 권위형 전투와 최대 4인 게임룸
- 20초 동일 세션 재접속, 입력 정지 보호와 실패 시 새 세션·로컬 훈련 폴백

- 회수→교전→보스→추출→디브리핑으로 이어지는 첫 작전 `OPERATION ZERO`
- 데이터 추적→교전→중계기 절단→보스→추출로 이어지는 두 번째 작전 `OP-01 ASHFALL`

- 가격·상품 효용·모집 확률·천장을 함께 공개하는 인게임 보급소
- 서버 영수증 검증 경계, 전 계정 거래 ID 중복 방지, 창립 보급 1회 지급

- 기본 비활성 선택 분석, AI 처리 고지, JSON 내보내기와 계정 삭제 개인정보 센터
- 모션 감소·UI 크기·적록 보정·고대비 접근성 설정

- 적·자원별 전술 실루엣, 총구 화염·피격·처치·폭풍·추출 VFX
- 외부 음원 없이 동작하는 합성 전투 SFX, 진동, 모션 감소 설정
- 첫 실행 개인정보 선택과 9단계 온보딩, 언제든 다시 볼 수 있는 튜토리얼
- 빌드·서버 버전 진단, 동의 기반 오류 큐와 30초 중복 억제
- 저장소 준비 상태 API와 8개 이상 동시 클라이언트 알파 부하 스모크
- 최대 128클라이언트·1시간 설정이 가능한 준비 상태/스냅샷 소크 테스트
- 서명 AAB 또는 디버그 APK를 생성하는 수동 Android 알파 워크플로
- 알파 빌드 기본 결제 비활성화와 테스터 피드백 진입점
- PWA 설치 아이콘 및 Capacitor 기반 Android 내부 테스트 프로젝트

> 딥 토크는 API 키 없이도 규칙 기반 페르소나 코어로 동작합니다. `OPENAI_API_KEY`가 서버에 설정되고 사용자가 별도로 동의한 경우에만 외부 AI 링크를 사용합니다. 제공자 키는 클라이언트에 포함하지 않으며, 실시간 전투 판정은 AI 응답과 분리되어 있습니다.

## 빠른 실행

Node.js 22 이상이 필요합니다.

```bash
npm install
npm run dev:all
```

`dev:all`은 클라이언트와 메모리 저장 방식의 게임서버를 함께 실행합니다. 브라우저 HUD의 서버 표시가 초록색으로 바뀌면 권위형 게임룸에 연결된 상태입니다.

PostgreSQL·Redis까지 포함한 전체 백엔드는 Docker로 실행합니다.

```bash
cp .env.example .env
# .env의 JWT_SECRET을 32자 이상의 무작위 값으로 설정
docker compose -f infra/docker-compose.yml up --build -d
npm run dev
```

Docker 서버를 시작하려면 `JWT_SECRET`을 반드시 설정해야 합니다. 공개 환경에 배포하기 전에는 `POSTGRES_PASSWORD`, `CORS_ORIGIN`도 운영값으로 바꾸세요. `.env.example`에는 비밀값을 넣지 않습니다.

외부 AI 딥 토크를 알파에서 시험할 때만 서버 환경에 `OPENAI_API_KEY`를 넣습니다. 기본 모델과 호출·안전 상한은 `OPENAI_MODEL`, `AI_DAILY_TURN_LIMIT`, `AI_TIMEOUT_MS`, `AI_MODERATION_ENABLED`로 조정합니다. 키가 없거나 제공자가 응답하지 않아도 로컬 페르소나 코어가 대화를 이어갑니다.

프로덕션 빌드와 핵심 시스템 테스트:

```bash
npm run check
npm run build
npm run build:server
```

Android 내부 테스트 프로젝트를 동기화하고 디버그 APK를 만들려면 Android SDK/JDK 17 환경에서 실행합니다.

```bash
npm run mobile:sync
cd android
./gradlew assembleDebug
```

APK는 `android/app/build/outputs/apk/debug/app-debug.apk`에 생성됩니다. 실서버를 연결하는 배포 빌드는 `VITE_GAME_SERVER_URL=https://...`를 지정한 뒤 동기화해야 하며, Play Console 업로드용 AAB에는 별도의 릴리스 키 서명과 개인정보·결제·데이터 안전성 설정이 필요합니다.

실행 중인 서버의 HTTP·WebSocket 통합 확인:

```bash
npm run smoke:server -- http://localhost:2567
npm run smoke:alpha -- http://localhost:2567 8 10000
npm run smoke:soak -- http://localhost:2567 16 120000
```

## 조작

| 기능 | PC | 모바일 |
|---|---|---|
| 이동 | `WASD` / 방향키 | 화면 방향 패드 |
| 조준/사격 | 마우스 클릭 유지 | `FIRE` 유지 시 자동 조준 |
| 긴급 회피 | `Space` | `DODGE` |
| 무기 전환 | 숫자 `1`·`2`·`3` | 화면 하단 무기 버튼 |
| 뉴럴 링크 | 게이지 100%에서 `Q` | 게이지 100%에서 리더 초상화 |
| 화물 추출 | 중앙 리프트에서 `E` | 중앙 리프트에서 `EXTRACT` |
| 동료 명령 | 하단 전술 입력창 | 하단 전술 입력창 |
| 쉘터/로스터 | 하단 메뉴 | 하단 메뉴 |
| 보급소 | 하단 `보급소` | 하단 `보급소` |

표준 게임패드는 왼쪽/오른쪽 스틱 이동·조준, A/RT 사격, B 회피, X 추출, Y 뉴럴 링크를 지원합니다.

명령 예시:

```text
방패병 시선 끌어줘, 내가 우회할게
모두 내 쪽으로 복귀해
루멘, 지금 치료해줘
저격수에게 집중 사격
```

## 구조

```text
├── src/                        # Phaser PC/모바일 클라이언트
│   └── game/network/           # Colyseus SDK 및 서버 동기화
├── server/
│   ├── src/rooms/              # 최대 4인 권위형 게임룸
│   ├── src/simulation/         # 이동·사격·적·자원·추출 판정
│   ├── src/api/                # 인증·프로필·경제 HTTP API
│   ├── src/economy/            # 모집·쉘터·방치·추출 트랜잭션
│   ├── src/commerce/           # 서버 영수증 검증과 상품 지급
│   ├── src/ai/                 # 동의·안전 필터·외부 AI 중계·규칙 폴백
│   └── src/persistence/        # 메모리/PostgreSQL 저장 어댑터
├── packages/shared/            # 클라이언트-서버 공용 프로토콜·전투·월드 기하
├── public/assets/operators/    # 오퍼레이터별 최적화 WebP 일러스트
├── android/                    # Capacitor Android 네이티브 프로젝트
└── infra/                       # Dockerfile 및 Compose
```

## 서버가 직접 판정하는 항목

- 입력 순서와 이동 벡터 검증, 월드 경계·하드 커버 충돌
- 회피 거리와 1.8초 재사용 대기시간, 구조물 관통 방지
- 발사 쿨다운, 조준각, 사거리, 엄폐 시야, 명중 및 피해
- 안전한 적 스폰·추적·엄폐 우회·공격
- 뉴럴 재머의 원거리 교란과 링크 게이지 감소
- 뉴럴 링크 충전·발동 조건과 역할별 전투 효과
- 방사능 폭풍, 사망, 현장 화물 소실
- 자원 습득과 추출 지점 검증
- 추출 재화의 1회성 DB 확정
- 작전 해금 순서, 작전별 게임룸 매칭과 최초 완료 보상의 1회성 DB 확정
- 모집 확률·SSR 천장·중복 보상
- 쉘터 비용과 최대 8시간 방치 보상
- 전술 장비 제작 비용·소유권·2슬롯 구성과 전투 보너스
- 플랫폼 영수증, 전역 거래 ID, 상품 1회성, 구독 만료와 지급

클라이언트는 입력만 보내고 결과를 임의로 확정할 수 없습니다. 경제 요청은 `Idempotency-Key`로 중복 실행을 방지하고 PostgreSQL에서는 플레이어 행을 잠근 트랜잭션으로 처리합니다.

상세 설계는 [Android 알파 운영서](docs/ALPHA_TESTING.md), [게임 디자인](docs/GAME_DESIGN.md), [AI/서비스 아키텍처](docs/ARCHITECTURE.md), [개인정보 초안](docs/PRIVACY.md), [출시 체크리스트](docs/RELEASE_CHECKLIST.md), [캐릭터 아트 가이드](docs/CHARACTER_ART.md), [월 300만원 실험 계획](docs/MONETIZATION.md), [출시 로드맵](docs/ROADMAP.md)을 참고하세요.

## 크로스플랫폼 전략

- Web/PWA: 현재 Vite 빌드를 그대로 배포
- Android: Capacitor 셸과 전용 아이콘·스플래시 적용, 결제·푸시·마이크 권한 어댑터는 출시 전 추가
- iOS: Capacitor 프로젝트, Apple 로그인·결제·마이크 권한 어댑터 추가 필요
- Steam/PC: Electron 또는 Tauri 셸, Steamworks 어댑터와 키보드/게임패드 옵션 추가
- 장기 3D 제품화: 게임 규칙·AI 서비스 계약을 유지한 채 Godot/Unity 클라이언트로 전환 가능

## 개발 원칙

- 전투 결과를 LLM 지연이나 환각에 의존시키지 않습니다. LLM은 의도 해석과 캐릭터 표현을 담당하고, 실제 행동은 검증된 전술 명령으로 제한합니다.
- 감정적 유대를 과금 압박에 사용하지 않습니다. 기억 보존·안전 설정·기본 대화는 무료 핵심 기능으로 유지합니다.
- 서버 권위형 경제와 입력 검증을 기본값으로 유지하고, 연령/지역별 확률형 아이템 규정 준수를 출시 전 필수 조건으로 둡니다.

> v1.2는 결제를 숨긴 비공개 알파입니다. 실제 스토어 공개 출시에는 운영자 정보가 포함된 공개 개인정보처리방침 URL, Google/Apple 결제 어댑터, 연령 등급, 실기기 QA, 외부 AI 실키 평가·레드팀과 고객지원 연락처가 추가로 필요합니다.
